# Автоматизация запуска

## Windows Task Scheduler (планировщик задач Windows)

### Создание задачи:

1. Откройте "Планировщик заданий" (Task Scheduler)
2. Нажмите "Создать основную задачу" (Create Basic Task)
3. Укажите имя и описание задачи
4. Выберите расписание (например, "Ежедневно" или "Еженедельно")
5. В разделе "Действие" выберите "Запуск программы"
6. В поле "Программа или сценарий" укажите путь к node.exe:
   ```
   C:\Program Files\nodejs\node.exe
   ```
7. В поле "Аргументы" укажите:
   ```
   C:\путь\к\вашему\проекту\PRODUCTION_READY\src\main.js
   ```
8. В поле "Начать в" укажите путь к папке проекта:
   ```
   C:\путь\к\вашему\проекту\PRODUCTION_READY
   ```

### Альтернативный способ через npm:
Вместо прямого запуска node, можно использовать npm:
- Программа: `C:\Windows\System32\cmd.exe`
- Аргументы: `/c npm start`
- Начать в: `C:\путь\к\вашему\проекту\PRODUCTION_READY`

## Linux/macOS Cron

### Для добавления в cron:

1. Откройте терминал
2. Введите `crontab -e` для редактирования crontab
3. Добавьте строку в формате:
   ```
   0 9 * * * cd /path/to/your/PRODUCTION_READY && /usr/bin/npm start
   ```
   Это запустит публикацию каждый день в 9 утра

### Примеры расписаний:
- Каждый день в 9 утра: `0 9 * * *`
- Каждый понедельник в 10 утра: `0 10 * * 1`
- Каждые 2 часа: `0 */2 * * *`
- Каждый день в 12 и 18 часов: `0 12,18 * * *`

## Использование пакетного файла (Windows)

Создайте файл `run_publisher.bat` в корне проекта:

```batch
@echo off
cd /d "C:\путь\к\вашему\проекту\PRODUCTION_READY"
npm start
pause
```

Затем можно запускать этот файл вручную или через планировщик задач.

## Использование bash-скрипта (Linux/macOS)

Создайте файл `run_publisher.sh`:

```bash
#!/bin/bash
cd /path/to/your/PRODUCTION_READY
npm start
```

Сделайте его исполняемым:
```bash
chmod +x run_publisher.sh
```

## Рекомендации по автоматизации

### Частота запуска
- Не запускайте слишком часто (рекомендуется 1-2 раза в день)
- Учитывайте время публикации (лучше утром или вечером)
- Делайте перерывы между публикациями (например, 1-2 часа)

### Мониторинг
- Проверяйте логи выполнения
- Убедитесь, что куки не истекают
- Проверяйте, что feed.xml содержит новые статьи

### Обработка ошибок
- Система отслеживания публикаций предотвращает дубликаты
- При ошибках публикации скрипт продолжит работу с следующей статьей
- Проверяйте файл published_articles.txt для отслеживания прогресса

## Примеры cron-выражений

- Каждый день в 8:00 утра: `0 8 * * *`
- Каждый будний день в 9:30: `30 9 * * 1-5`
- Каждый первый день месяца в 10:00: `0 10 1 * *`
- Каждые 6 часов: `0 */6 * * *`
- Каждую пятницу в 17:00: `0 17 * * 5`

## Важные замечания

- Убедитесь, что Node.js доступен в PATH
- Проверьте права доступа к файлам проекта
- Убедитесь, что компьютер/сервер включен в нужное время
- Рассмотрите возможность использования systemd-сервиса для Linux (для более надежной автоматизации)
