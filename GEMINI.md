# 🧘 ZenMaster v7.1 (Stable) - Project Context

Automated AI Content Factory for Yandex.Dzen, specialized in high-quality storytelling and authentic visual generation.

## 🏗 Project Overview

ZenMaster is a modular TypeScript/Node.js system designed to produce viral content for Yandex.Dzen at scale. It leverages Google Gemini for narrative generation and image synthesis, with specialized post-processing to ensure platform-native authenticity.

### Core Architecture
- **Orchestration**: `ContentFactoryOrchestrator` manages the lifecycle from theme to export.
- **Narrative Engine**: `GeminiService` uses multi-stage generation (Plan -> Hook -> Development -> Climax -> Resolution) with RAG (style injection).
- **Human-Centric Restoration**: `TextRestorationService` bypasses AI detectors not through broken syntax, but through high-quality "human-like" flow: varying sentence lengths (rhythm of breathing), sensory details, and an engaging conversational tone ("Show, don't tell").
- **Visual Intelligence**: `ImageGeneratorAgent` extracts story context to generate unique cinematic covers, followed by `MobilePhotoAuthenticityProcessor` which applies mobile-camera filters based on the story's emotional tone.
- **Automation**: A robust pipeline (GitHub Actions) handles the full lifecycle: Generate -> Restore -> Feed -> Publish.
- **Monitoring & Alerts**: Every stage has built-in error reporting. If a workflow fails, it sends an email to `sheepoff@gmail.com` and creates an error log in the repository root for AI diagnosis.

## 🛠 Monitoring & Troubleshooting

### Error Logs
If a process fails, look for these files in the root directory:
- `ERROR_REPORT.md`: Fatal errors during article generation.
- `RESTORE_ERROR.md`: Failures during the text restoration stage.
- `FEED_ERROR.md`: Issues with RSS feed generation (blocks publishing).
- `publish-*.log`: Detailed Playwright logs for Dzen publishing attempts.

**Action for AI**: If any of these files exist, read them immediately to diagnose the failure and propose a fix.

## 🚀 Key Commands

### Content Generation
- **Generate RAW + RESTORED Pairs (Default)**:
  ```bash
  node --import tsx cli.ts both --count=1 --channel=ethno-food-ritual --images
  ```
- **Mass Generation (Factory Mode)**:
  ```bash
  node --import tsx cli.ts factory --count=5 --preset=medium-batch --images
  ```

### Validation & Tools
- **Validate Articles**: `npm run validate` (Scores articles based on quality and AI-detection).
- **Restore Articles**: `node scripts/restore-articles-safe.js path/to/article.md` (Manual restoration trigger).
- **Generate RSS Feed**: `npm run feed:full` or `npm run feed:incremental`.
- **Auto-Publish**: `node --import tsx scripts/publish-dzen.ts` (Publishes the next article from feed.xml).

## 📂 Project Structure

- `cli.ts`: Main entry point for the system.
- `services/`: Core logic (Orchestrator, Gemini, Image Gen, Sanitizer, Playwright).
- `config/`: Channel configurations (`dzen-channels.config.ts`) and global settings.
- `articles/`: Output directory for generated Markdown and images.
- `scripts/`: Utility scripts for feed generation, restoration, and publishing.

## 📏 Development Conventions
- **Human-Centric Style**: Avoid artificial word order (inversions) or excessive particles. Focus on "Rhythm of Breathing" and sensory details to bypass AI detectors.
- **Show, Don't Tell**: Every story must use active verbs and specific details rather than abstract adjectives.
- **Services**: All core logic must reside in `services/` and follow the established orchestrator/worker pattern.
- **Consistency**: Ensure the mascot "Батон" (The Baton Guard) is correctly referenced as the constant companion.
- **Authenticity**: Every image must pass through `MobilePhotoAuthenticityProcessor` to remove AI signatures and add mobile-like artifacts.
- **Environment**: Sensitive keys (Gemini API, cookies) are stored in `.env` and GitHub Secrets.
